# MemeFlow 国内可用方案 — 双轨架构规划

> 创建日期：2026-04-30 凌晨
> 状态：**规划中**，未开工

---

## 一句话目标

**让中国大陆用户在不开 VPN 的情况下，能用上 memeflow 的全部功能**——翻译、梗百科、课堂同传、单词本、复习。

---

## 为什么国内现在用不了 — 5 层障碍

```
用户 → [1.域名解析] → [2.静态资源] → [3.登录] → [4.AI 调用] → [5.课堂同传]
        ↓             ↓             ↓          ↓           ↓
      可能被墙       Firebase慢      Google OAuth  Gemini API   Deepgram WS
                                  国内打不开    国内不通      国内不稳
```

**任何一层不通，整个产品就不能用**。所以必须每一层都解决。

---

## 整体架构（双轨并行）

```
┌────────────────────────────────┐  ┌────────────────────────────────┐
│   memeflow.com (海外版-现版)     │  │   memeflow-cn.com (国内版-新)   │
├────────────────────────────────┤  ├────────────────────────────────┤
│ 域名      Firebase Hosting       │  │ 域名      阿里云 OSS+CDN+备案   │
│ 登录      Google OAuth          │  │ 登录      微信 / 手机号短信      │
│ 数据库    Firestore             │  │ 数据库    阿里云 MongoDB / MySQL │
│ 函数      Cloud Functions       │  │ 函数      阿里云函数计算 FC      │
│ AI        Gemini 2.5/3 Pro     │  │ AI        DeepSeek / 通义 / 豆包 │
│ 同传 ASR  Deepgram nova-3      │  │ 同传 ASR  腾讯云 / 讯飞          │
│ 付费      Stripe               │  │ 付费      微信支付 / 支付宝       │
│ 错误监控   Sentry               │  │ 错误监控   阿里云日志服务 SLS     │
└────────────────────────────────┘  └────────────────────────────────┘
              ↓                                  ↓
              └──── 共享代码仓库（编译时切环境）─────┘
```

**核心思想**：一份代码，**编译时**根据环境变量决定走海外组件还是国内组件。不是运行时切换、不是用户手动选——根据访问域名自动走对应栈。

## 6 个关键决策点

### 决策 1：域名 + 备案（最关键，必须最先开始）

| 选项 | 含义 | 备案时长 | 限制 |
|------|------|---------|------|
| A. 公司主体备案 | 用公司营业执照备案 | 7-15 天 | 任何业务都行 |
| B. 个人备案 | 用身份证备案 | 15-30 天 | 不能做交易/支付 |
| C. 香港服务器（不备案） | 阿里云香港 ECS | 0 天 | 速度比备案版慢 30-50%，但能立刻用 |

**我推 A** 如果你有公司主体；没有就 B；急的话 C 是过渡方案。

**问你**：
- 你有公司主体吗？
- 国内版要不要做付费？（决定能不能用 B）

### 决策 2：账号体系

| 选项 | 海外用户访问国内域名 | 国内用户访问海外域名 | 开发难度 |
|------|---------------------|---------------------|---------|
| A. 双账号独立 | 重新注册 | 重新注册 | 简单（1 周） |
| B. 账号互通 | 自动登录 | 自动登录 | 难（2-3 周） |
| C. 国内版只支持国内登录 | 不支持 | 用 Google 登录 | 中等（1-2 周） |

**我推 A**：实际上海外用户和国内用户基本是两批人；强行互通会引入跨境数据合规问题（GDPR + 国内数据法）。

**问你**：你的目标用户是同一批人，还是真的国内独立做一个市场？

## 风险清单（**这些是我担心的**）

1. **数据迁移**：Firestore → MongoDB/RDS 数据结构差异大，可能要重写一半 model 层。
2. **登录体系**：微信登录有自己的官方流程（appid/secret + 公众号绑定），不像 Google OAuth 那样标准。学习成本 3-5 天。
3. **法律合规**：国内有数据安全法、个人信息保护法。用户隐私政策需要重写。**v1 不包含跨境数据，规避大部分问题**，但仍要写中文版隐私协议。
4. **质量回归**：DeepSeek 输出风格跟 Gemini 不一样，需要对所有 prompt 做一轮调优测试。可能 1-2 周。
5. **课堂同传适配**：腾讯云 ASR 跟 Deepgram 协议完全不同，重新写 liveSession.ts 大半部分。这是最大的不确定性，所以 v1 砍掉。
